Amazon Web Services (AWS) is a cloud services platform, offering compute power, database storage, content delivery and other functionality that enables high availability, durability and scalability. AWS also strongly emphasizes security, with services that respond to DDoS attacks, provide encryption in transit and at rest, and monitoring for an entire environment.

The two services we will focus on today will be S3 (Simple Storage Service), and EC2 (Elastic Compute Cloud).

S3 is a storage service that can store an umlimited amount of objects that can each range from a size of 0 bytes to 5 terabytes. Objects are stored in 'buckets' which can be configured with various security policies.
EC2 is infrastructure as a service (IaaS) that allows users to create virtual machines to deploy applications, function as a database, or utilized for high-performance computing.

When a web page is served, it has to point to an html file that specifies how the page should look like in this case, we will use it as confirmation that our host is internet facing, being able to be accessed by others

Within AWS, it is possible to serve our index.html in multiple ways.

In this workshop, we will host it on S3 and EC2, highlighting the use cases and differences between both services.
